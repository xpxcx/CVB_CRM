## 1. Краткое описание API

Веб-API **MiniCRM Booking** реализовано на **Node.js** и **Express** (практическая работа №3). Сервис предоставляет:

- **GET** — проверка работоспособности, список услуг, список записей;
- **POST** `/api/slots/suggest` — расчёт свободных временных слотов по окну поиска, длительности, буферу и переданным занятым интервалам;
- **POST** `/api/appointments` — создание записи с данными клиента (хранение в памяти процесса в учебной версии).

Исходный код: репозиторий на GitHub (ссылка: *указать после публикации*).  
Развёрнутая версия: *URL базового адреса после деплоя, например `https://....amvera.io`*.

## 2. Шаги развёртывания (облако)

Ниже — универсальная схема; конкретную платформу выбрать по заданию (**Amvera**, **Render**, **Railway**, **Azure** и т.п.).

### 2.1. Подготовка репозитория GitHub

1. Создать репозиторий на GitHub (публичный или приватный с доступом для платформы).
2. В каталоге проекта `MiniCRM_API` выполнить: `git init` (если ещё не), `git add .`, `git commit -m "MiniCRM API"`, привязать `remote` и `git push -u origin main` (или `master`).
3. Убедиться, что в корне есть `package.json`, `server.js`, `slotEngine.js`; для Docker-деплоя — `Dockerfile` (в проекте добавлен).

### 2.2. Вариант A: Amvera (по заданию)

1. Зарегистрироваться на [Amvera](https://amvera.ru/) и выбрать **тестовый тарифный план** (по заданию — рисунок 1).
2. Создать новый проект, **подключить репозиторий GitHub**, указать ветку и корень репозитория (если API в подпапке — указать путь к `MiniCRM_API`).
3. Тип приложения: **контейнер (Docker)** или **Node.js** — в зависимости от мастера Amvera: если Dockerfile — собрать образ; если Node — указать команду запуска `npm start` и переменную `PORT` (платформа часто задаёт порт автоматически).
4. Переменные окружения: `PORT` обычно выставляет хостинг; при необходимости задать вручную в панели.
5. Дождаться сборки и деплоя, скопировать **публичный URL** приложения.

### 2.3. Вариант B: Render / Railway (бесплатные тиры)

1. Подключить GitHub-репозиторий.
2. **Build:** `npm install` (или Docker, если используется Dockerfile).
3. **Start:** `node server.js` или `npm start`.
4. Сохранить выданный HTTPS-URL.

### 2.4. Docker (локальная проверка образа)

```bash
docker build -t minicrm-api .
docker run -p 3000:3000 minicrm-api
```

После деплоя подставить в Postman базовый URL вместо `http://localhost:3000`.

## 3. План тестирования

Подробный план с эндпоинтами, методами, телами запросов и ожидаемыми кодами — в файле **`ПЛАН_ТЕСТИРОВАНИЯ_API.md`** в том же репозитории.

Кратко проверяемые маршруты:

| Эндпоинт | Метод | Назначение |
|----------|-------|------------|
| `/api/health` | GET | Health-check |
| `/api/services` | GET | Список услуг |
| `/api/appointments` | GET | Список записей |
| `/api/slots/suggest` | POST | расчёт слотов |
| `/api/appointments` | POST | создание записи |
| `/api/unknown` | GET | проверка 404 |
