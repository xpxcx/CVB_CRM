# План тестирования API MiniCRM Booking

**Базовый URL (локально):** `http://localhost:3000`  
**Базовый URL (после деплоя):** `https://<ваш-домен>/` — подставить из облака.

Критерий успеха: код ответа и структура JSON совпадают с ожидаемыми, нет 500 при корректных данных, время отклика приемлемо (< 3 с на тестовом тарифе).

---

## 1. GET /api/health

| Поле | Значение |
|------|----------|
| Метод | GET |
| URL | `/api/health` |
| Тело | нет |
| Ожидаемый код | 200 |
| Ожидаемый JSON (пример) | `{"status":"ok","service":"MiniCRM Booking API","version":"1.0.0"}` |

---

## 4. POST /api/slots/suggest (корректный запрос)

| Поле | Значение |
|------|----------|
| Метод | POST |
| URL | `/api/slots/suggest` |
| Headers | `Content-Type: application/json` |
| Тело (JSON) | См. ниже |
| Ожидаемый код | 200 |
| Ожидаемый результат | `slots` — массив объектов с `start`, `end`, `score`, `reason`; `meta.ok === true` |

Пример тела:

```json
{
  "durationMinutes": 60,
  "windowStart": "2026-04-05T09:00:00.000Z",
  "windowEnd": "2026-04-05T18:00:00.000Z",
  "bufferMinutes": 10,
  "busyIntervals": [
    { "start": "2026-04-05T12:00:00.000Z", "end": "2026-04-05T13:00:00.000Z" }
  ]
}
```

---

## 5. POST /api/slots/suggest (ошибка: неверное окно)

| Поле | Значение |
|------|----------|
| Метод | POST |
| URL | `/api/slots/suggest` |
| Тело | `{"durationMinutes":60,"windowStart":"2026-04-05T18:00:00.000Z","windowEnd":"2026-04-05T09:00:00.000Z"}` (конец раньше начала) |
| Ожидаемый код | 200 |
| Ожидаемый результат | `slots: []`, `meta.ok: false` |

---

## 6. POST /api/appointments (корректный запрос)

| Поле | Значение |
|------|----------|
| Метод | POST |
| URL | `/api/appointments` |
| Headers | `Content-Type: application/json` |
| Тело (JSON) | См. ниже |
| Ожидаемый код | 201 |
| Ожидаемый результат | `data` с полями `id`, `clientName`, `start`, `end`, `status`, `createdAt` |

Пример тела:

```json
{
  "clientName": "Тестовый клиент",
  "clientPhone": "+79001234567",
  "serviceId": "svc-1",
  "start": "2026-04-10T10:00:00.000Z",
  "end": "2026-04-10T11:00:00.000Z"
}
```

---

## 7. POST /api/appointments (ошибка: нет обязательных полей)

| Поле | Значение |
|------|----------|
| Метод | POST |
| URL | `/api/appointments` |
| Тело | `{}` или без `clientName` / `start` / `end` |
| Ожидаемый код | 400 |
| Ожидаемый JSON (пример) | `{"error":"Укажите clientName, start, end ..."}` |
