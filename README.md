# MiniCRM Booking — веб-API

## 1. Название и назначение

**MiniCRM Booking API** — RESTful веб-сервис для учебного проекта по дисциплине «Цифровые возможности для бизнеса». Сервис предназначен для сценария **записи клиентов к мастеру/репетитору**: подбор свободных временных слотов и создание записей.

## 2. Решаемая задача и целевая аудитория

**Задача:** автоматизировать подбор времени для записи на услугу и фиксацию заявок без полноценной CRM — в объёме лабораторных работ (ПР 1–4).

**Аудитория:** разработчики и тестировщики учебного проекта; конечный пользователь интерфейса (клиент/мастер) в данной версии не реализован отдельным фронтендом — взаимодействие через HTTP-клиенты (Postman, Swagger UI, curl).

## 3. Основные функции

| Функция | Описание |
|--------|----------|
| Проверка работоспособности | `GET /api/health` — статус сервиса и версия API. |
| Справочник услуг | `GET /api/services` — демонстрационный список услуг (название, длительность, цена). |
| Список записей | `GET /api/appointments` — записи, созданные через POST (в памяти процесса). |
| Подбор слотов | `POST /api/slots/suggest` — расчёт списка свободных интервалов по окну дат, длительности, буферу между записями и массиву занятых интервалов `busyIntervals` (имитация данных календаря). |
| Создание записи | `POST /api/appointments` — сохранение записи с именем клиента, временем начала/окончания, опционально телефоном и услугой. |
| Документация API | `GET /api-docs` — **Swagger UI** (OpenAPI 3). Сырой JSON спецификации: `GET /openapi.json`. |

## 4. Архитектура и технологии

| Компонент | Технология |
|-----------|------------|
| Среда выполнения | Node.js 18+ (ES Modules) |
| HTTP-сервер | Express.js |
| Бизнес-логика подбора слотов | модуль `slotEngine.js` (чистые функции) |
| Хранение данных | оперативная память процесса (массивы; после перезапуска данные теряются) |
| База данных | в учебной версии **не используется**; при необходимости можно подключить PostgreSQL/SQLite без смены контрактов API |
| Документация | OpenAPI 3 в файле `openapi.json`, отображение через **swagger-ui-express** |
| Контейнеризация | Dockerfile (для Render, Amvera и др.) |

Структура репозитория:

- `server.js` — точка входа, маршруты, Swagger UI
- `slotEngine.js` — расчёт слотов
- `openapi.json` — описание эндпоинтов для Swagger
- `package.json` — зависимости

## 5. Развёртывание и запуск

### Локально

```bash
npm install
npm start
```

По умолчанию порт **3000** (или переменная окружения `PORT`).

- API: `http://localhost:3000`
- **Swagger UI:** `http://localhost:3000/api-docs`
- OpenAPI JSON: `http://localhost:3000/openapi.json`

### Облако (Render и аналоги)

1. Подключить репозиторий GitHub.
2. Команда сборки: `npm install` (или образ из Dockerfile).
3. Команда запуска: `npm start` или `node server.js`.
4. Задать переменную `PORT`, если платформа не подставляет её автоматически (Render обычно подставляет).

После деплоя откройте в браузере: `https://<ваш-домен>/api-docs` и проверьте «Try it out» для эндпоинтов.

### Docker

```bash
docker build -t minicrm-api .
docker run -p 3000:3000 minicrm-api
```

## 6. Проверка Swagger UI

1. Запустить сервер.
2. Перейти на `/api-docs`.
3. Убедиться, что отображаются теги, пути и схемы запросов/ответов.
4. Выполнить тестовый запрос через **Try it out** (для POST указать JSON-тело из примеров).
